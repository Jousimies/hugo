Emacs 以及 org-mode

Emacs 是什么？官网对其的解释是：一个可扩展的、可定制的、免费自由的文本编辑器，以及更多的可能性。

An extensible, customizable, free/libre text editor — and more.

文本编辑器有很多，如 Vim，Vscode，至于为什么选 Emacs，有如下几点：

  • Emacs 是一种流行且功能强大的文本编辑器，已被广泛使用四十年。最初是为 Unix 操作系统开发的，但现在可用于许多其他平台，包括 Windows 和 macOS。

  • Emacs 的关键特性之一是定制性和可扩展性，允许用户向编辑器添加自己的功能和命令。Emacs目前己有40多年的历史，目前仍处于极积开发当中，生命力非常旺盛。Emacs 具有一个庞大而充满活力的用户社区，开发了广泛的插件和扩展，为编辑器添加了额外的功能。Emacs-china 是当前最好的Emacs 中文论坛。

  • Emacs 能够处理各种基于文本的任务。除了作为一个通用的文本编辑器，它还可以用于编程、电子邮件、新闻阅读等。

  • 最重要的一点是 Emacs 有 Org-mode 这一强大应用，而且是独家，其他编辑器上实现的 org-mode 都不完整。或许你知道 Markdown，但是不同的编辑器实现的 Markdown 略有区别，相比 org-mode，还是存在着不足。

总的来说，Emacs 是一个备受推崇和广泛使用的文本编辑器，因其多功能性、定制性和效率而受到重视。无论是程序员、作家，还是只是每天与文本打交道的人，Emacs 绝对值得考虑作为文本编辑器。

那么什么是org-mode 呢？官网的解释是：Org-mode是Emacs中的一个 major mode, 一个快速高效的纯文本系统，用于记笔记、编写文档、计算笔记、文学编程、维护任务列表、规划项目以及更多。

A GNU Emacs major mode forkeeping notes, authoring documents, computational notebooks, literate programming, maintaining to-do lists, planning projects, and more — in a fast and effective plain text system.

以及什么是 org-roam:

Org-roam is a tool for networked thought. It reproduces some of Roam Research’s key features withinOrg-mode.

  • 笔记系统

我对于笔记管理软件的总体需求是：​本地储存，纯文本，格式可控，非商业软件 。

传统的笔记工具以纸笔为载体，各种手账、剪纸等，以纸笔为载体是纸媒时代的最有效的进行笔记管理的方式，如何有效整理信息这本书的作者就提出了一元化的笔记方法，作者本人也积累了大量的笔记本。个人很配服他的这种毅力，平常人很难一以贯之的坚持进行笔记的记录。

但是我个人不怎么喜欢这种基于纸笔的笔记管理方式，当前是网络时代，我们有太多的信息是基于网站等获取的，我们身处于电子时代，我认为笔记的最终去处是电子化。以电子笔记为主，辅之纸笔。出门不一定会携带纸笔，但是一定会带上手机，部份情况下不适合使用电子设备进行笔记的记录也可以暂时的使用纸笔进行记录，事后再进行电子化。

现在电子笔记软件有太多的可选项：Onenote 、印象笔记、Roam Research 、Obsidian、Logseq、Notion、Mediawiki、Tiddly wiki、蚂蚁笔记、Simplenote、Joplin 等等。可选择的太多有很多人就有了困难选择症。我使用过的笔记软件或系统有很多，上面列举的软件，都有轻度和深度使用过。

Mediawiki，Tiddly wiki 是基于 html 组织页面。Mediawiki 需要自己有一些计算机相关的知识，搭建系统。Tiddly wiki相对简单点。这两者维护起来对于纯小白来说有点超纲，使用不方便。我个人认为个人笔记上这类系统有点得不尝失。

Onenote 、Notion 等是专有格式，意味着当需要夸平台流转笔记时，其专有格式会带来一些麻烦。

Onenote、印象笔记、Notion等笔记软件存在的问题是它们是富文本编辑软件，离开了软件本身，笔记内容无法查看。相对富文本而言，纯文本格式的笔记内容就不存在上面的问题。

我不使用 Obsidian 或 Logseq 的原因是它们都属于商业软件。商业软件的功能是它提供啥你用啥，有什么个性化的需求，自己较难定制。此外有些商业软件不提供自定义的文件存储位置，放置在他的服务器上，可靠性不能保证，如wolai。不想自己的笔记给别人窥探吧？！

对于纯文本笔记的要求，markdown 和 org-mode 都可以满足，这两种语言之上都有着相对应的软件实现。

Org-mode 相比 markdown ，特性要丰富很多， markdown 相对较简陋。 org-mode 写完之后可以根据需要将其转化为其他的格式，如 html，docx，latex 等等。

以一生作为个人笔记的保存时长，长久的、可控的保存自己的笔记，Emacs 是理想的软件。使用 Emacs 进行笔记管理完美符合对于笔记软件的总本需求，基于 Emacs 编辑器构建自己的笔记系统，JUST DO IT !!!

笔记内容

记笔记很难的一点在于归类，以什么主题进行记录。如如何有效整理信息一书的作者提出的不分来源，不分重要性，只要是自己关心的都可以记录下来。

不分来源，不分重要性，只要是自己关心、想记录下来的内容，都可以一点一滴地写在笔记本上。《如何有效整理信息》

在 GTD 这本书中，作者提出了无压工作这一概念。记笔记也是如此，怎么做到无压记笔记？那就是把文字写下来就好，不用在意格式，文法，快速的记下当时当下脑海中的想法是笔记管理的第一步。

这样也可以解决记笔记真的有用吗？的疑问。笔记记录的作用不一定在当下，有可能在以后，当你需要的时候有一个系统可以供你查询。

关于笔记的主题或者分类我的观点就是不分类，或者只是给一个大类而不具体的进行区分，当相关的笔记积累了足够多的内容再进行整理。不依赖分类，取而代之的是建立以搜索为基础的笔记管理方法（下文会说如何搜索）。

关于All-in-one 笔记系统的几点要求，列举如下：

以笔记系统为中心，集成资讯、阅读、视频、音频的碎片化知识管理；笔记系统中通过网状进行链接。

笔记管理的主体内容：格式、文本、图、表、视频、音频、文献参考资料管理等等。

文本与格式

要做到无压记录笔记，第一点是要做到文本与格式的分离。记笔记的主体内容就是写下文字内容，而不是生成花里胡哨的格式。这类笔记如Onenote、印象笔记。

文本与格式分离，这不是哪个软件都能做好的。如 word 就很难做到样式和内容的分离，以我的经验，它可以做到但是很难，你需要对word 有充分的了解。Latex 就很容易做到文本与样式的分离，但是它难在非所见即所得。（有什么软件你需要深度使用，但是可以对其不需要了解的吗？没有什么是纯傻瓜式的。）

图与表格

图和表格是笔记系统中一个很重要的组成部分。

笔记系统中插入图有两种方式，一种是如word ，印象笔记的方式。一种是类似org-mode, markdown这种方式，将图以链接的方式插入到笔记当中。我选择第二种。

图直接插入到笔记当中的方式，往往是专有格式的软件。这类笔记系统对于软件本体依赖过高，不利于笔记在不同系统中的转移。图直接插入到笔记当中，会导致单个笔记的体积很大，而且很难在不同的笔记中进行复用。相应的使用图片链接的方式就没有上述的问题。

在 org-mode 中不建议把图片放置到笔记所在位置，而是使用一个统一的文件位置放置。这样当你想找一个图片的时候就有两种方式，一是联想图片相关的笔记内容，进行全文的搜索。二是打开图片的位置进行查找。如果你放置到笔记所在的位置，那么图片增加后怎么查找就很难进行。

Org-mode 中管理图片，可以使用模板系统插入或者使用 org-download 插件将剪切板或网页中的图片直接插入org 笔记中。会自动生成下面的语句，其中 Name 和 caption 的内容需要自己写，不写也没事。只在导出到html、pdf时有用。

#+NAME: fig: Emacs Makes Amending Configurations Simple
#+CAPTION: Emacs Makes Amending Configurations Simple
#+ATTR_ORG: :width 500px
#+ATTR_LaTeX: :width 12cm :placement [!htpb]
#+ATTR_HTML: :width 600px

关于表格，我个人的意见是笔记系统中不要使用过于复杂的表格。复杂的表格内容，使用EXCEL进行处理，或者使用CSV格式保存使用Python等语言编程处理。

这里要提到 onenote 笔记软件，它的页面就像一个画板，笔记如果想要内容规整，需要嵌套使用表格（可以很漂亮）。页面像画板当内容超出屏幕范围时，很难对内容有个全局的了解。 我认为笔记系统中不适合保存过于复杂的表格数据，过于复杂的表格用Excel这类软件处理。

Latex 公式

笔记系统中的公式我只推荐使用 latex ，以文本的方式进行管理。

如 Word 上使用 mathtype 进行公式的输入非常差，难以编排，交叉引用。

如果你认为latex 输入公式很慢，不直观，是大概率你没有使用合适的方法。你需要的是快速输入的方法以及公式预览。Latex公式的效果看看这个 Demo。

如果你需要将公式插入到word中，需要进行转换。

打开链接，将Latex公式粘入，点击convert，转换成xml格式的，在其头部加入下面的语句，整体复制到word中即可。

<?xml version="1.0"?>

如果只有个别公式，那么你在word 中输入，也没什么痛点。但是如果你有很多的公式要处理，那么使用latex生成，再通过上述的转换要轻松很多。

如果你是现成的公式需要转latex，可以使用mathtool这个工具，刚出来的时候使用还是免费的，目前有次数限制，你也可以注册开发者获取api，好像有1000次使用。

使用latex手敲也没什么痛点啦，打的可以很快的，我觉的比手写快。

文献管理

文献管理软件有很多，如zotero，endnote，mendeley，jabref等等。我更加建议使用 Zotero 及浏览器插件进行，使用 BetterBib生成 bibtex 文件供 Emacs 调用。

文献文件的格式，大部份应该都是pdf 文件。如果是caj等格式，那么推荐你使用tampermonkey在知网上下载pdf 格式的，而不是caj格式的（我讨厌caj）。如果你看的是出版物，网上有很多提供的epub格式。

RSS(RDF Site Summary or Really Simple Syndication)is a web feed that allows users and applications to access updates to websites in a standardized, computer-readable format.

知乎上同样有很多关于 RSS的话题，感兴的话可以搜索看看。科研期刊、计算机相关博客等有很好用的 RSS 源，其他的需要使用 rsshub 生成。

笔记工作流

使用 Emacs 的一个很大的好处是可以行成自己的工作流。假如你要阅读文献，记录笔记，对于不认识的单词或者语句进行查询和翻译。

在 Emacs 中可以通过快捷键调用 Zotero 生成的 Bibtex 文件，打开 bibtex 中记录的 pdf 文件进行阅读，然后你认为需要对该文件进行笔记的记录，可以在阅读 pdf 的界面调用相关命令生成笔记文件，生成笔记文件的同时可以自动插入笔记模版。开两个并列的 buffer，一边阅读一边进行记录。根据需要可以将笔记的内容与 pdf 文件的页码对应上。

你阅读的同时在pdf 文件上作了很多的标记，你想要把标记的内容提取到笔记当中，也可以通过相关的命令一键提取到文献笔记当中。

在 pdf 界面或者笔记记录界面调用翻译引擎进行翻译。

你做了一个笔记，想起来其他的笔记与本笔记的内容相关，你可以直接在本笔记中引用相关的文献进行记录，或者直接将相关文献的笔记内容引用进来（区别于复制粘贴的方法）。

当你记录了很多的相关的笔记，引用了很多的文献，你可以在文献引用上回车打开相应的 pdf 文件或者参考文献笔记文件。相比较使用word进行文献记录，你还需要一个个去查找。

Emacs 的不足之处

  • Emacs是三大平台可用的，但是在Win上使用有点不好。

    体验的话，Linux上体验最好、性能也最好，如eaf。Mac上体验也很好，相对不好的是win。

    Win上由于文件系统的原因对于小文件的操作性能不足；有些包在win上也不可用；启动速度有些慢等。

    Win上目前有wsl系统，也可以使用emacs，等微软把Linux子系统这个搞的好用些，使用Emacs也就没什么问题了。

  • 移动端的使用体验不好。 Android上可以使用orgzly、Termux等。

    2023-02-24，目前Android可以运行 Emacs 软件，见 Froid.

    Iphone上有Beorg等软件。

    以上不要指望和电脑上一样的体验。
